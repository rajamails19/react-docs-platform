# useCallback

`useCallback` returns a memoized version of a function that only changes if one of its dependencies changes. It prevents child components from re-rendering when the parent re-renders but the function hasn't changed.

```tsx
const memoizedFn = useCallback(() => {
  doSomething(a, b)
}, [a, b])
```

## The Problem: New Function Every Render

```tsx
// ❌ Without useCallback — handleClick is a NEW function on every parent render
function Parent() {
  const [count, setCount] = useState(0)
  const [name, setName] = useState('')

  // New object reference every render → ExpensiveChild always re-renders
  const handleClick = () => {
    console.log('clicked')
  }

  return (
    <div>
      <input value={name} onChange={e => setName(e.target.value)} />
      <ExpensiveChild onClick={handleClick} />
    </div>
  )
}

// React.memo won't help if onClick changes every render
const ExpensiveChild = React.memo(({ onClick }: { onClick: () => void }) => {
  console.log('ExpensiveChild rendered')
  return <button onClick={onClick}>Click</button>
})
```

Every time you type in the input, `Parent` re-renders. A new `handleClick` is created. Even though `React.memo` wraps `ExpensiveChild`, the prop changed (new function reference) so it re-renders too.

## The Fix: useCallback

```tsx
// ✅ With useCallback — handleClick is stable across renders
function Parent() {
  const [count, setCount] = useState(0)
  const [name, setName] = useState('')

  const handleClick = useCallback(() => {
    console.log('clicked')
  }, [])  // empty deps — never changes

  const handleCountClick = useCallback(() => {
    setCount(c => c + 1)   // use updater form to avoid count in deps
  }, [])

  return (
    <div>
      <input value={name} onChange={e => setName(e.target.value)} />
      <ExpensiveChild onClick={handleClick} />
    </div>
  )
}
```

Now when you type, `handleClick` is the same reference — `ExpensiveChild` doesn't re-render.

## With Dependencies

```tsx
function SearchPage() {
  const [query, setQuery]   = useState('')
  const [filters, setFilter] = useState({ category: 'all' })

  // Recreates only when query or filters changes
  const handleSearch = useCallback(async () => {
    const results = await searchAPI(query, filters)
    setResults(results)
  }, [query, filters])

  return (
    <>
      <SearchBar onChange={setQuery} onSearch={handleSearch} />
      <FilterPanel onChange={setFilter} onApply={handleSearch} />
    </>
  )
}
```

## When NOT to Use useCallback

`useCallback` has a cost: it holds a reference, runs a dependency comparison on every render, and adds code complexity. Only use it when:

1. The function is passed to a child wrapped in `React.memo`
2. The function is in a dependency array of `useEffect`, `useMemo`, or another `useCallback`

```tsx
// ❌ Pointless — this function isn't passed down or used in deps
function Counter() {
  const [count, setCount] = useState(0)

  // No child component, no deps — useCallback adds overhead for zero benefit
  const increment = useCallback(() => setCount(c => c + 1), [])

  return <button onClick={increment}>{count}</button>
}

// ✅ Correct — without useCallback, just inline it
function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>
}
```

## useCallback vs useMemo

```tsx
// useCallback — memoizes a FUNCTION
const memoFn = useCallback(() => compute(x), [x])

// useMemo — memoizes the RESULT of a function
const memoValue = useMemo(() => compute(x), [x])

// They're equivalent:
useCallback(fn, deps) === useMemo(() => fn, deps)
```

## Summary

<Callout type="success" title="What we learned">
- `useCallback` memoizes functions so they have stable references across renders
- Functions are recreated each render — without `useCallback`, `React.memo` on children is ineffective
- Only use it when the function is passed to `React.memo` children or used in `useEffect`/`useMemo` deps
- Use the updater pattern `setCount(c => c + 1)` to remove state from the dependency array
- `useCallback(fn, deps)` is equivalent to `useMemo(() => fn, deps)`
</Callout>
