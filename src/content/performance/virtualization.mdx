# List Virtualization

List virtualization (also called "windowing") is the technique of rendering only the DOM nodes that are currently visible in the viewport, plus a small buffer. As the user scrolls, nodes outside the viewport are unmounted and replaced by new ones entering from the other side.

<Callout type="info" title="The core insight">
A list of 10,000 items does not need 10,000 DOM nodes. At any given moment the user can see perhaps 15–20 rows. Virtualization keeps the DOM at a constant size regardless of data length.
</Callout>

## Why It Matters

### The DOM Node Problem

Every DOM node costs memory and CPU time. When React renders a list of 10,000 rows:

- 10,000 `<div>` elements are created in the DOM
- The browser lays out and paints all of them (even those off-screen)
- React holds 10,000 fiber nodes in memory
- Scroll events trigger style recalculations across the entire list

```
Without virtualization (10,000 rows):
  Initial render: ~4,200ms
  Memory usage:   ~380MB
  Scroll FPS:     12–18fps (janky)

With virtualization (10,000 rows, ~15 visible):
  Initial render: ~85ms
  Memory usage:   ~22MB
  Scroll FPS:     60fps (smooth)
```

## react-window

`react-window` is the most widely-used virtualization library. It provides four components:

| Component | Use case |
|---|---|
| `FixedSizeList` | All rows are the same height |
| `VariableSizeList` | Rows have different heights |
| `FixedSizeGrid` | Uniform cell size in both axes |
| `VariableSizeGrid` | Variable row heights and/or column widths |

### Installation

```bash
npm install react-window
npm install --save-dev @types/react-window
```

### FixedSizeList — Step-by-Step Implementation

```tsx
import { FixedSizeList, ListChildComponentProps } from 'react-window'
import { memo } from 'react'

interface Employee {
  id: string
  name: string
  department: string
  email: string
  salary: number
}

// Step 1: Extract the row as a separate memoized component.
// IMPORTANT: The row component receives `index`, `style`, and `data`
// from react-window. The `style` prop MUST be applied — it contains
// the absolute positioning that places the row correctly.
const EmployeeRow = memo(function EmployeeRow({
  index,
  style,
  data,
}: ListChildComponentProps<Employee[]>) {
  const employee = data[index]

  // Alternate row background for readability
  const bgColor = index % 2 === 0 ? '#ffffff' : '#f9fafb'

  return (
    // style is required — contains top/height/width/position
    <div
      style={{ ...style, backgroundColor: bgColor }}
      className="employee-row"
      role="row"
      aria-rowindex={index + 1}
    >
      <span className="col-name">{employee.name}</span>
      <span className="col-dept">{employee.department}</span>
      <span className="col-email">{employee.email}</span>
      <span className="col-salary">${employee.salary.toLocaleString()}</span>
    </div>
  )
})

// Step 2: Render the list
function EmployeeList({ employees }: { employees: Employee[] }) {
  return (
    <div role="table" aria-label="Employee Directory" aria-rowcount={employees.length}>
      {/* Header row — outside the virtual list */}
      <div className="header-row" role="row">
        <span>Name</span>
        <span>Department</span>
        <span>Email</span>
        <span>Salary</span>
      </div>

      <FixedSizeList
        height={600}         // Visible height of the list container (px)
        itemCount={employees.length}  // Total number of items
        itemSize={56}        // Height of each row in px
        width="100%"         // Width of the list container
        itemData={employees} // Passed as `data` to every row — stable reference!
        overscanCount={5}    // Extra rows to render above/below viewport
      >
        {EmployeeRow}
      </FixedSizeList>
    </div>
  )
}
```

<Callout type="danger" title="Never forget the style prop">
Every row component rendered by react-window receives a `style` prop containing `position: absolute`, `top`, `height`, and `width`. If you omit `style` from your row's root element, all rows will stack on top of each other at position 0. This is the most common react-window mistake.
</Callout>

### VariableSizeList — Dynamic Row Heights

Use `VariableSizeList` when rows have different heights (e.g., expandable rows, varying text length).

```tsx
import { VariableSizeList } from 'react-window'
import { useRef, useCallback } from 'react'

interface Message {
  id: string
  author: string
  body: string
  attachmentCount: number
}

// Pre-calculate or estimate heights
function getItemHeight(messages: Message[], index: number): number {
  const message = messages[index]
  // Base height + extra for attachments
  const baseHeight = 72
  const attachmentHeight = message.attachmentCount * 32
  // Estimate text lines (~60 chars per line, 20px per line)
  const textLines = Math.ceil(message.body.length / 60)
  const textHeight = textLines * 20
  return baseHeight + attachmentHeight + textHeight
}

function MessageThread({ messages }: { messages: Message[] }) {
  const listRef = useRef<VariableSizeList>(null)

  // Wrap in useCallback for stable reference
  const getHeight = useCallback(
    (index: number) => getItemHeight(messages, index),
    [messages]
  )

  // When content changes and heights shift, reset the cache
  const resetHeightCache = useCallback(() => {
    listRef.current?.resetAfterIndex(0)
  }, [])

  return (
    <VariableSizeList
      ref={listRef}
      height={800}
      itemCount={messages.length}
      itemSize={getHeight}  // Function, not a number
      width="100%"
      itemData={messages}
      estimatedItemSize={100} // Helps initial layout calculation
    >
      {({ index, style, data }) => (
        <MessageRow message={data[index]} style={style} />
      )}
    </VariableSizeList>
  )
}
```

## Overscan — Pre-Rendering Buffer

The `overscanCount` prop tells react-window how many extra items to render beyond the visible viewport. This prevents flashes of empty content during fast scrolling.

```
Viewport (shows items 10–25):
┌─────────────────────────────┐
│ overscan: items 5–9         │  ← rendered but invisible (above)
├─────────────────────────────┤
│ VISIBLE: items 10–25        │  ← visible to user
├─────────────────────────────┤
│ overscan: items 26–30       │  ← rendered but invisible (below)
└─────────────────────────────┘
  items 1–4 and 31+: unmounted
```

```tsx
<FixedSizeList
  overscanCount={3}   // Default: 1 (often too low for fast scrolling)
  // ...
>
```

<Callout type="tip" title="Tuning overscanCount">
A value of 3–5 is good for most cases. Higher values reduce blank flashes during fast scrolling but increase the number of rendered DOM nodes. On very fast machines or with simple row components, 2 may be sufficient.
</Callout>

## FixedSizeGrid — Two-Dimensional Virtualization

Spreadsheet-like UIs need virtualization on both axes:

```tsx
import { FixedSizeGrid, GridChildComponentProps } from 'react-window'

interface CellData {
  matrix: number[][]
  headers: string[]
}

const Cell = memo(function Cell({
  columnIndex,
  rowIndex,
  style,
  data,
}: GridChildComponentProps<CellData>) {
  if (rowIndex === 0) {
    // Header row
    return (
      <div style={style} className="grid-header">
        {data.headers[columnIndex]}
      </div>
    )
  }

  const value = data.matrix[rowIndex - 1][columnIndex]
  return (
    <div
      style={style}
      className={`grid-cell ${value < 0 ? 'negative' : 'positive'}`}
    >
      {value.toLocaleString()}
    </div>
  )
})

function DataGrid({ matrix, headers }: CellData) {
  const cellData: CellData = { matrix, headers }

  return (
    <FixedSizeGrid
      columnCount={headers.length}
      columnWidth={120}       // Each column is 120px wide
      rowCount={matrix.length + 1}  // +1 for header row
      rowHeight={40}          // Each row is 40px tall
      height={500}            // Visible height
      width={800}             // Visible width
      itemData={cellData}
    >
      {Cell}
    </FixedSizeGrid>
  )
}
```

## TanStack Virtual (react-virtual)

TanStack Virtual v3 takes a headless approach — it provides a hook that gives you measurements and positions, but you build the DOM yourself. This makes it far more flexible for custom layouts, CSS frameworks, and non-standard scroll containers.

### Installation

```bash
npm install @tanstack/react-virtual
```

### Basic Implementation

```tsx
import { useVirtualizer } from '@tanstack/react-virtual'
import { useRef } from 'react'

function VirtualizedUserList({ users }: { users: User[] }) {
  // ref to the scrollable container element
  const parentRef = useRef<HTMLDivElement>(null)

  const virtualizer = useVirtualizer({
    count: users.length,           // Total number of items
    getScrollElement: () => parentRef.current, // The scroll container
    estimateSize: () => 56,        // Estimated item height in px
    overscan: 5,                   // Extra items above/below viewport
  })

  return (
    // Scrollable container
    <div
      ref={parentRef}
      style={{ height: '600px', overflow: 'auto' }}
    >
      {/* Total height spacer — makes the scrollbar correct */}
      <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
        {virtualizer.getVirtualItems().map(virtualItem => (
          <div
            key={virtualItem.key}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              height: `${virtualItem.size}px`,
              transform: `translateY(${virtualItem.start}px)`,
            }}
          >
            <UserRow user={users[virtualItem.index]} />
          </div>
        ))}
      </div>
    </div>
  )
}
```

### Variable Size with Measurement

TanStack Virtual can measure actual rendered sizes, avoiding the need to pre-calculate heights:

```tsx
function DynamicHeightList({ posts }: { posts: Post[] }) {
  const parentRef = useRef<HTMLDivElement>(null)

  const virtualizer = useVirtualizer({
    count: posts.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 100,    // Initial estimate
    measureElement: (element) => element.getBoundingClientRect().height,
  })

  return (
    <div ref={parentRef} style={{ height: '800px', overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
        {virtualizer.getVirtualItems().map(virtualItem => (
          <div
            key={virtualItem.key}
            // ref callback triggers measureElement
            ref={virtualizer.measureElement}
            data-index={virtualItem.index}
            style={{
              position: 'absolute',
              top: 0,
              left: 0,
              width: '100%',
              transform: `translateY(${virtualItem.start}px)`,
            }}
          >
            {/* Height is NOT set here — measured automatically */}
            <PostCard post={posts[virtualItem.index]} />
          </div>
        ))}
      </div>
    </div>
  )
}
```

## Infinite Scroll with Virtualization

Infinite scroll + virtualization is a common pattern for social feeds and data tables. The list grows as the user scrolls to the bottom, while old items remain virtualized.

```tsx
import { useVirtualizer } from '@tanstack/react-virtual'
import { useRef, useEffect, useCallback } from 'react'
import { useInfiniteQuery } from '@tanstack/react-query'

const PAGE_SIZE = 50

function InfinitePostFeed() {
  const parentRef = useRef<HTMLDivElement>(null)

  const {
    data,
    fetchNextPage,
    hasNextPage,
    isFetchingNextPage,
  } = useInfiniteQuery({
    queryKey: ['posts'],
    queryFn: ({ pageParam = 0 }) =>
      api.getPosts({ offset: pageParam, limit: PAGE_SIZE }),
    getNextPageParam: (lastPage, pages) =>
      lastPage.hasMore ? pages.length * PAGE_SIZE : undefined,
  })

  // Flatten paginated data into a single array
  const posts = data?.pages.flatMap(page => page.items) ?? []
  // Add a sentinel item for the loading indicator
  const totalItems = hasNextPage ? posts.length + 1 : posts.length

  const virtualizer = useVirtualizer({
    count: totalItems,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 120,
    overscan: 5,
  })

  const virtualItems = virtualizer.getVirtualItems()
  const lastItem = virtualItems[virtualItems.length - 1]

  // Fetch next page when the sentinel item enters the viewport
  useEffect(() => {
    if (!lastItem) return
    if (lastItem.index >= posts.length - 1 && hasNextPage && !isFetchingNextPage) {
      fetchNextPage()
    }
  }, [lastItem, posts.length, hasNextPage, isFetchingNextPage, fetchNextPage])

  return (
    <div ref={parentRef} style={{ height: '100vh', overflow: 'auto' }}>
      <div style={{ height: virtualizer.getTotalSize(), position: 'relative' }}>
        {virtualItems.map(virtualItem => {
          const isLoaderRow = virtualItem.index >= posts.length

          return (
            <div
              key={virtualItem.key}
              style={{
                position: 'absolute',
                top: 0,
                left: 0,
                width: '100%',
                height: `${virtualItem.size}px`,
                transform: `translateY(${virtualItem.start}px)`,
              }}
            >
              {isLoaderRow ? (
                isFetchingNextPage ? (
                  <LoadingSpinner />
                ) : (
                  <div>No more posts</div>
                )
              ) : (
                <PostCard post={posts[virtualItem.index]} />
              )}
            </div>
          )
        })}
      </div>
    </div>
  )
}
```

## Accessibility Considerations

Virtual lists present accessibility challenges because most content is not in the DOM. These are the key mitigations:

### ARIA Roles and Attributes

```tsx
function AccessibleVirtualList({ items }: { items: Item[] }) {
  return (
    <div
      role="listbox"
      aria-label="Item list"
      aria-rowcount={items.length}  // Total count, not visible count
      // Omit aria-activedescendant for now — managed separately
    >
      {/* Header */}
      <div role="rowgroup">
        <div role="row">
          <span role="columnheader">Name</span>
          <span role="columnheader">Status</span>
        </div>
      </div>

      {/* Virtual rows */}
      <FixedSizeList
        height={600}
        itemCount={items.length}
        itemSize={48}
        width="100%"
        itemData={items}
      >
        {({ index, style, data }) => (
          <div
            style={style}
            role="option"
            aria-rowindex={index + 1}  // 1-based per ARIA spec
            aria-selected={false}
            tabIndex={-1}
          >
            {data[index].name}
          </div>
        )}
      </FixedSizeList>
    </div>
  )
}
```

### Keyboard Navigation

Virtual lists require custom keyboard handling because DOM focus is lost when items unmount:

```tsx
function KeyboardNavigableList({ items }: { items: Item[] }) {
  const listRef = useRef<FixedSizeList>(null)
  const [focusedIndex, setFocusedIndex] = useState(0)

  const handleKeyDown = useCallback(
    (e: React.KeyboardEvent) => {
      if (e.key === 'ArrowDown') {
        e.preventDefault()
        const next = Math.min(focusedIndex + 1, items.length - 1)
        setFocusedIndex(next)
        listRef.current?.scrollToItem(next, 'smart')
      } else if (e.key === 'ArrowUp') {
        e.preventDefault()
        const prev = Math.max(focusedIndex - 1, 0)
        setFocusedIndex(prev)
        listRef.current?.scrollToItem(prev, 'smart')
      } else if (e.key === 'Home') {
        setFocusedIndex(0)
        listRef.current?.scrollToItem(0)
      } else if (e.key === 'End') {
        const last = items.length - 1
        setFocusedIndex(last)
        listRef.current?.scrollToItem(last)
      }
    },
    [focusedIndex, items.length]
  )

  return (
    <div
      role="listbox"
      aria-activedescendant={`item-${focusedIndex}`}
      tabIndex={0}
      onKeyDown={handleKeyDown}
    >
      <FixedSizeList
        ref={listRef}
        height={500}
        itemCount={items.length}
        itemSize={48}
        width="100%"
        itemData={{ items, focusedIndex }}
      >
        {({ index, style, data }) => (
          <div
            id={`item-${index}`}
            style={style}
            role="option"
            aria-selected={index === data.focusedIndex}
            className={index === data.focusedIndex ? 'focused' : ''}
          >
            {data.items[index].name}
          </div>
        )}
      </FixedSizeList>
    </div>
  )
}
```

<Callout type="warning" title="Screen reader limitations">
Virtual lists remain challenging for screen readers. NVDA and JAWS may not correctly announce the total count or navigate off-screen items. For accessibility-critical applications, consider pagination as an alternative — paginated lists render fully in the DOM and work reliably with all assistive technologies.
</Callout>

## Scrolling to a Specific Item

Both libraries expose an imperative scroll API:

```tsx
// react-window
const listRef = useRef<FixedSizeList>(null)

// Scroll modes: 'auto' | 'smart' | 'center' | 'end' | 'start'
listRef.current?.scrollToItem(42, 'center')

// TanStack Virtual
const virtualizer = useVirtualizer({ ... })
virtualizer.scrollToIndex(42, { align: 'center' })
```

```tsx
// Practical: "Jump to my item" button
function ListWithJumpTo({ items, highlightedId }: Props) {
  const listRef = useRef<FixedSizeList>(null)

  const jumpToHighlighted = useCallback(() => {
    const index = items.findIndex(item => item.id === highlightedId)
    if (index !== -1) {
      listRef.current?.scrollToItem(index, 'smart')
    }
  }, [items, highlightedId])

  return (
    <>
      <button onClick={jumpToHighlighted}>Jump to selected item</button>
      <FixedSizeList ref={listRef} /* ... */ >
        {/* rows */}
      </FixedSizeList>
    </>
  )
}
```
