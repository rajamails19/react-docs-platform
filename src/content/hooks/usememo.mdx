# useMemo

`useMemo` memoizes the **result** of a computation. It only recalculates when its dependencies change, preventing expensive work from running on every render.

```tsx
const memoizedValue = useMemo(() => expensiveComputation(a, b), [a, b])
```

## The Problem It Solves

Every render re-executes the entire component function body. Expensive operations inside become a performance problem:

```tsx
function ProductList({ products, category, sortBy }) {
  // ❌ Recalculates on EVERY render — even unrelated state changes
  const filtered = products
    .filter(p => p.category === category)
    .sort((a, b) => a[sortBy] - b[sortBy])
    .slice(0, 100)

  return <ul>{filtered.map(p => <ProductRow key={p.id} product={p} />)}</ul>
}
```

If `ProductList` re-renders because of a tooltip hover or a parent rerender, this filter+sort runs unnecessarily.

## The Fix: useMemo

```tsx
function ProductList({ products, category, sortBy }) {
  // ✅ Only recalculates when products, category, or sortBy changes
  const filtered = useMemo(() =>
    products
      .filter(p => p.category === category)
      .sort((a, b) => a[sortBy] - b[sortBy])
      .slice(0, 100),
    [products, category, sortBy]
  )

  return <ul>{filtered.map(p => <ProductRow key={p.id} product={p} />)}</ul>
}
```

## Real-World Analogy

useMemo is like a **TurboTax filing you've already done**.

- First year: you do all the work (compute taxes)
- Same income, deductions, filing status: use last year's copy
- Something changed: redo only the relevant section

## Measuring With useMemo

```tsx
import { useMemo, useRef } from 'react'

function ExpensiveComponent({ data }: { data: number[] }) {
  const renderCount = useRef(0)

  const result = useMemo(() => {
    const start = performance.now()

    // Simulate expensive operation
    const computed = data.reduce((acc, val) => {
      let res = val
      for (let i = 0; i < 10000; i++) res = Math.sqrt(res + i)
      return acc + res
    }, 0)

    const duration = performance.now() - start
    console.log(`Computed in ${duration.toFixed(2)}ms`)

    return computed
  }, [data])

  renderCount.current++

  return (
    <div>
      <p>Result: {result.toFixed(4)}</p>
      <p>Renders: {renderCount.current}</p>
    </div>
  )
}
```

## Stable References for Child Components

useMemo is critical when passing objects or arrays to `React.memo`-wrapped children:

```tsx
function Parent({ userId }: { userId: string }) {
  const [theme, setTheme] = useState('light')

  // ❌ New object reference on every render → MemoChild always re-renders
  const config = { userId, role: 'admin' }

  // ✅ Stable reference → MemoChild only re-renders when userId changes
  const config = useMemo(() => ({ userId, role: 'admin' }), [userId])

  return (
    <div>
      <button onClick={() => setTheme(t => t === 'light' ? 'dark' : 'light')}>
        Theme: {theme}
      </button>
      <MemoChild config={config} />  {/* Won't re-render on theme change */}
    </div>
  )
}

const MemoChild = React.memo(({ config }: { config: Config }) => {
  console.log('MemoChild rendered')
  return <div>{config.userId}</div>
})
```

## useMemo vs useCallback

| | `useMemo` | `useCallback` |
|---|---|---|
| Memoizes | Return value of a function | The function itself |
| Use case | Computed values, transformed data | Event handlers, callbacks |
| Example | `useMemo(() => data.filter(...), [data])` | `useCallback(() => handleClick(), [dep])` |
| Equivalent | `useCallback(fn, deps)` === `useMemo(() => fn, deps)` | Same |

```tsx
// These are equivalent:
const fn = useCallback(() => doSomething(a), [a])
const fn = useMemo(() => () => doSomething(a), [a])
```

## Common Mistakes

```tsx
// ❌ Wrapping cheap operations
const doubled = useMemo(() => count * 2, [count])   // trivial — no benefit

// ❌ Wrong dependencies (stale data)
const filtered = useMemo(() =>
  products.filter(p => p.active),
  []  // missing [products] → filtered is stale when products updates!
)

// ❌ Using useMemo as a side-effect mechanism
const _ = useMemo(() => {
  analytics.track('view', { page })   // ← this is a side effect, use useEffect!
  return null
}, [page])
```

## Summary

<Callout type="success" title="What we learned">
- `useMemo` caches expensive computed values and only recalculates when dependencies change
- Use it for truly expensive operations (filtering/sorting large arrays, complex math)
- Critical for **stable references** when passing objects to `React.memo` children
- Don't over-use — the overhead isn't worth it for cheap computations
- `useCallback` is `useMemo` for functions
</Callout>
