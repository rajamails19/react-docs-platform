These questions test deep understanding of React internals, architectural patterns, performance optimization, and the ability to make informed trade-offs at scale.

<Callout type="tip" title="How to approach senior interviews">
Senior interviews are about **trade-offs and reasoning**, not just knowing answers. Explain *why* you'd choose one approach over another, mention what you'd consider before making a decision, and demonstrate that you've encountered real production problems.

## Architecture & System Design Questions

These open-ended questions test your ability to reason about large systems:

### Q: How would you build a real-time collaborative text editor in React?

**Approach:**
1. Use **operational transforms** or **CRDTs** (e.g., Yjs, Automerge) for conflict-free concurrent edits
2. **WebSocket** for real-time sync; **HTTP SSE** as fallback
3. **Optimistic updates** — apply local edits instantly, sync in background
4. **Presence indicators** — show cursors/selections of other users
5. **Offline support** — queue edits in IndexedDB, sync on reconnect
6. Editor core: **ProseMirror** or **Lexical** (Meta's editor)

### Q: Design the state management strategy for a large SaaS application

**Layered approach:**
- **Server state**: RTK Query or React Query — caching, deduplication, background refresh
- **UI state**: component-local `useState` / `useReducer`
- **Shared UI state**: lightweight Zustand store (modals, notifications, theme)
- **Complex domain state**: Redux Toolkit with normalized entities
- **URL state**: React Router search params for filters/pagination (shareable URLs)
- **Form state**: React Hook Form — uncontrolled, highly performant
