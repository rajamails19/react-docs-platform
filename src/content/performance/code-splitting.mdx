# Code Splitting

Code splitting breaks your JavaScript bundle into smaller chunks that load on demand. Instead of sending 2MB of JavaScript on the first page load, you send only what's needed for the current view.

<Callout type="info" title="Why it matters">
Every additional 100KB of JavaScript adds ~1 second of parsing time on low-end mobile devices. Code splitting is one of the highest-ROI performance optimizations for React apps.
</Callout>

## The Problem: One Big Bundle

Without code splitting, Vite/Webpack puts everything into one bundle:

```
dist/
└── assets/
    └── index-Cx7ygL2H.js   // 2.4 MB — user downloads ALL of this
```

The user downloading your settings page also downloads the code for your reports, admin panel, and every other route — even if they never visit them.

## React.lazy + Suspense

`React.lazy` creates a dynamically-imported component:

```tsx
import { lazy, Suspense } from 'react'

// ❌ Eager import — always in the bundle
import Dashboard from './pages/Dashboard'
import Settings  from './pages/Settings'
import Reports   from './pages/Reports'

// ✅ Lazy imports — loaded on demand
const Dashboard = lazy(() => import('./pages/Dashboard'))
const Settings  = lazy(() => import('./pages/Settings'))
const Reports   = lazy(() => import('./pages/Reports'))

function App() {
  return (
    <Suspense fallback={<PageSkeleton />}>
      <Routes>
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/settings"  element={<Settings />} />
        <Route path="/reports"   element={<Reports />} />
      </Routes>
    </Suspense>
  )
}
```

Result:
```
dist/
└── assets/
    ├── index-Cx7.js          // 380 KB — initial bundle
    ├── Dashboard-D8k.js      //  95 KB — loaded on /dashboard
    ├── Settings-B3m.js       //  42 KB — loaded on /settings
    └── Reports-A7q.js        // 210 KB — loaded on /reports
```

## Granular Splitting Strategies

### Route-level splitting (most impactful)

```tsx
// All routes are lazy-loaded
const Home         = lazy(() => import('@/pages/Home'))
const UserProfile  = lazy(() => import('@/pages/UserProfile'))
const AdminDashboard = lazy(() => import('@/pages/admin/Dashboard'))  // large admin bundle
const Analytics    = lazy(() => import('@/pages/Analytics'))

// Wrap entire router in one Suspense
<Suspense fallback={<AppShell loading />}>
  <Routes>{/* ... */}</Routes>
</Suspense>
```

### Component-level splitting (heavy widgets)

```tsx
// Heavy third-party components
const RichTextEditor = lazy(() => import('./RichTextEditor'))   // includes ProseMirror
const DataGrid       = lazy(() => import('./DataGrid'))         // includes AG Grid
const PDFViewer      = lazy(() => import('./PDFViewer'))        // includes pdf.js

function DocumentEditor({ type }: { type: 'editor' | 'pdf' }) {
  return (
    <Suspense fallback={<div>Loading editor...</div>}>
      {type === 'editor' ? <RichTextEditor /> : <PDFViewer />}
    </Suspense>
  )
}
```

### Conditional feature splitting

```tsx
// Load admin features only for admin users
function App() {
  const { user } = useAuth()

  const AdminPanel = user.role === 'admin'
    ? lazy(() => import('./AdminPanel'))
    : null

  return (
    <Suspense fallback={null}>
      {AdminPanel && <AdminPanel />}
    </Suspense>
  )
}
```

## Prefetching

Start loading chunks before the user needs them:

```tsx
// Prefetch on hover (user likely to click)
function NavLink({ to, children }: { to: string; children: React.ReactNode }) {
  const prefetch = () => {
    // Dynamically import to start the download without rendering
    if (to === '/dashboard') import('./pages/Dashboard')
    if (to === '/reports')   import('./pages/Reports')
  }

  return (
    <Link to={to} onMouseEnter={prefetch}>
      {children}
    </Link>
  )
}
```

## Error Boundaries for Lazy Components

What if the lazy chunk fails to load (network error)?

```tsx
class ChunkErrorBoundary extends React.Component<
  { children: ReactNode; fallback: ReactNode },
  { hasError: boolean }
> {
  state = { hasError: false }

  static getDerivedStateFromError() {
    return { hasError: true }
  }

  componentDidCatch(error: Error) {
    // Log chunk load errors to Sentry
    if (error.name === 'ChunkLoadError') {
      Sentry.captureException(error)
      // Optional: auto-refresh to get fresh chunk
      window.location.reload()
    }
  }

  render() {
    if (this.state.hasError) return this.props.fallback
    return this.props.children
  }
}

// Usage
<ChunkErrorBoundary fallback={<ErrorPage />}>
  <Suspense fallback={<LoadingSpinner />}>
    <LazyRoute />
  </Suspense>
</ChunkErrorBoundary>
```

## Bundle Size Best Practices

```tsx
// ❌ Import entire lodash — 69KB
import _ from 'lodash'
const result = _.groupBy(items, 'category')

// ✅ Import only what you need — 1.5KB
import groupBy from 'lodash/groupBy'
const result = groupBy(items, 'category')

// ✅ Even better — use native equivalents
const result = Object.groupBy(items, item => item.category)  // ES2024

// ❌ Import entire date-fns — 80KB
import { format } from 'date-fns'

// ✅ Tree-shake correctly (date-fns v3 is ESM-first)
import { format } from 'date-fns'  // only format is bundled with proper bundler
```

## Summary

<Callout type="success" title="What we learned">
- Code splitting reduces initial bundle size by loading code only when needed
- `React.lazy()` + `Suspense` is the standard pattern for route and component splitting
- Always split at route boundaries first — highest impact per effort
- Prefetch on hover for near-instant navigation after first load
- Use bundle analysis tools to identify large dependencies and optimize them
</Callout>
