# useEffect

`useEffect` lets you synchronize a component with **external systems** — APIs, browser APIs, third-party libraries, timers, subscriptions. It runs *after* the browser has painted the screen.

```tsx
useEffect(() => {
  // side effect code
  return () => {
    // cleanup (optional)
  }
}, [dependencies])
```

## What Is a Side Effect?

A side effect is anything that **reaches outside the component's render scope**:
- Fetching data from an API
- Setting up a timer or interval
- Subscribing to browser events (resize, scroll, keydown)
- Reading/writing to localStorage
- Integrating with third-party libraries (maps, analytics, charts)
- Manually updating the DOM

<Callout type="info" title="Effects run after render">
React paints the UI first, then runs your effect. This ensures effects never block the visual update. For layout measurements that need to happen before paint, use `useLayoutEffect`.
</Callout>

## The Dependency Array

The second argument controls when the effect runs:

```tsx
// 1. No dependency array — runs after EVERY render
useEffect(() => {
  document.title = `Count: ${count}`
})

// 2. Empty array [] — runs ONCE after initial render
useEffect(() => {
  fetchUserData()
}, [])

// 3. With dependencies — runs when any dependency changes
useEffect(() => {
  fetchUser(userId)
}, [userId])  // re-runs only when userId changes
```

## Cleanup Functions

Always clean up effects that create subscriptions, timers, or listeners:

```tsx
useEffect(() => {
  // Setup
  const timer = setInterval(() => {
    setSeconds(s => s + 1)
  }, 1000)

  // Cleanup — runs before next effect or unmount
  return () => clearInterval(timer)
}, [])
```

### Why cleanup matters

Without cleanup:
- **Memory leaks** — interval keeps firing even after component unmounts
- **State updates on unmounted components** — causes React warnings and bugs
- **Stale subscriptions** — event listeners piling up

```tsx
// ✅ Complete effect lifecycle example
useEffect(() => {
  const controller = new AbortController()

  const fetchData = async () => {
    try {
      const res = await fetch(`/api/user/${userId}`, { signal: controller.signal })
      const data = await res.json()
      setUser(data)
    } catch (err) {
      if (err instanceof Error && err.name !== 'AbortError') {
        setError(err.message)
      }
    }
  }

  fetchData()

  return () => controller.abort()  // Cancel in-flight request on cleanup
}, [userId])
```

## Common Patterns

### Data fetching

```tsx
function UserProfile({ userId }: { userId: string }) {
  const [user, setUser]     = useState<User | null>(null)
  const [loading, setLoading] = useState(true)
  const [error, setError]   = useState<string | null>(null)

  useEffect(() => {
    let cancelled = false
    setLoading(true)

    fetchUser(userId)
      .then(data => {
        if (!cancelled) {
          setUser(data)
          setLoading(false)
        }
      })
      .catch(err => {
        if (!cancelled) {
          setError(err.message)
          setLoading(false)
        }
      })

    return () => { cancelled = true }
  }, [userId])

  if (loading) return <Spinner />
  if (error)   return <Error message={error} />
  return <UserCard user={user!} />
}
```

<Callout type="tip" title="Consider React Query instead">
For production apps, replace manual fetch effects with **React Query** (TanStack Query). It handles caching, deduplication, background refetch, loading/error states, and cancellation automatically.
</Callout>

### Event listeners

```tsx
function useWindowSize() {
  const [size, setSize] = useState({ width: window.innerWidth, height: window.innerHeight })

  useEffect(() => {
    const handleResize = () => {
      setSize({ width: window.innerWidth, height: window.innerHeight })
    }
    window.addEventListener('resize', handleResize)
    return () => window.removeEventListener('resize', handleResize)
  }, [])  // [] — register once, clean up on unmount

  return size
}
```

### Local storage sync

```tsx
function usePersist<T>(key: string, initial: T) {
  const [value, setValue] = useState<T>(() => {
    const stored = localStorage.getItem(key)
    return stored ? JSON.parse(stored) : initial
  })

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value))
  }, [key, value])

  return [value, setValue] as const
}
```

## Common Mistakes

```tsx
// ❌ Missing dependency
useEffect(() => {
  fetchUser(userId)  // uses userId but not listed
}, [])               // stale userId after prop changes!

// ✅ Include all dependencies
useEffect(() => {
  fetchUser(userId)
}, [userId])

// ❌ Async function directly in useEffect
useEffect(async () => {  // useEffect can't return a Promise
  const data = await fetch('/api')
}, [])

// ✅ Define async function inside
useEffect(() => {
  const run = async () => {
    const data = await fetch('/api')
  }
  run()
}, [])

// ❌ Object/array in dependency array (new reference every render)
useEffect(() => {
  fetchWithOptions(options)
}, [{ method: 'GET' }])  // new object every render → infinite loop!

// ✅ Use primitive values or useMemo/useCallback
const options = useMemo(() => ({ method: 'GET' }), [])
useEffect(() => {
  fetchWithOptions(options)
}, [options])
```

## useEffect vs useLayoutEffect

| | `useEffect` | `useLayoutEffect` |
|---|---|---|
| When it runs | After browser paint | Before browser paint (after DOM mutations) |
| Blocks paint? | No (async) | Yes (synchronous) |
| Use case | Data fetching, subscriptions, logging | DOM measurement, tooltip positioning, animations |
| SSR | Safe (no error) | Warns on server |

## Summary

<Callout type="success" title="What we learned">
- `useEffect` synchronizes a component with external systems (APIs, timers, subscriptions)
- The dependency array controls when it runs: always, once, or on specific changes
- Always return a **cleanup function** for subscriptions, timers, and event listeners
- Use `AbortController` for cancellable fetch requests
- For production data fetching, prefer **React Query** over manual effects
</Callout>
