# What is React?

React is an **open-source JavaScript library** for building user interfaces (UIs). It was created by Jordan Walke at Facebook in 2011, open-sourced in 2013, and is now maintained by Meta and a massive open-source community.

<Callout type="info" title="Library, not a framework">
React is a **library**, not a full framework. It focuses solely on the **View** layer (what the user sees). You choose your own routing, state management, and server-side tools.
</Callout>

## The Core Idea

At its heart, React gives you one superpower: **build UIs from reusable, composable components**.

Instead of manually manipulating the DOM like this:

```js
// Old way — jQuery-style imperative DOM manipulation
const button = document.createElement('button')
button.textContent = 'Click me'
button.addEventListener('click', () => {
  const count = parseInt(button.dataset.count || '0') + 1
  button.textContent = `Clicked ${count} times`
  button.dataset.count = String(count)
})
document.body.appendChild(button)
```

React lets you describe **what the UI should look like** for any given state:

```tsx
// React way — declarative
function Counter() {
  const [count, setCount] = React.useState(0)

  return (
    <button onClick={() => setCount(count + 1)}>
      Clicked {count} times
    </button>
  )
}
```

React figures out the minimum DOM changes needed when state changes. You describe **what**, React handles **how**.

## Real-World Analogy

Think of React like **Lego bricks**.

Each brick (component) is:
- **Self-contained** — it knows its own shape and behavior
- **Composable** — bricks snap together to build complex structures
- **Reusable** — the same brick appears in multiple places
- **Replaceable** — swap one brick without rebuilding everything

Facebook's News Feed, Instagram's photo grid, Airbnb's listing cards — all built from Lego bricks snapped together.

## Key Characteristics

| Concept | What it means |
|---|---|
| **Declarative** | Describe *what* the UI looks like, not *how* to build it |
| **Component-Based** | Build encapsulated pieces that manage their own state |
| **Learn Once, Write Anywhere** | React for web, React Native for mobile — same mental model |
| **Unidirectional Data Flow** | Data flows down through props; events bubble up |
| **Virtual DOM** | React diffs a virtual tree instead of touching real DOM every frame |

## What React Is NOT

- ❌ Not a complete framework (no built-in routing, HTTP client, or ORM)
- ❌ Not a replacement for your server (it's a client-side UI library)
- ❌ Not opinionated about styling (use CSS modules, Tailwind, styled-components — your choice)
- ❌ Not tied to the browser (React Native, React Three Fiber, Remotion — React targets many renderers)

## Try It — Your First React Component

Edit the code and see React render your changes instantly:

<LiveEditor code={`import { useState } from 'react'

// A component is just a function that returns JSX
function Greeting({ name }: { name: string }) {
  return (
    <div style={{ fontSize: 20, fontWeight: 700, color: '#6366f1' }}>
      Hello, {name}! 👋
    </div>
  )
}

// useState makes the UI reactive — React re-renders when state changes
function Counter() {
  const [count, setCount] = useState(0)
  return (
    <div style={{ display: 'flex', alignItems: 'center', gap: 12, marginTop: 12 }}>
      <button
        onClick={() => setCount(c => c - 1)}
        style={{ width: 36, height: 36, borderRadius: 8, border: '1px solid #e5e7eb', cursor: 'pointer', fontSize: 18, background: '#f9fafb' }}
      >−</button>
      <span style={{ fontSize: 24, fontWeight: 800, minWidth: 40, textAlign: 'center' }}>{count}</span>
      <button
        onClick={() => setCount(c => c + 1)}
        style={{ width: 36, height: 36, borderRadius: 8, border: '1px solid #e5e7eb', cursor: 'pointer', fontSize: 18, background: '#f9fafb' }}
      >+</button>
    </div>
  )
}

// Components compose — build complex UIs from small pieces
export default function App() {
  return (
    <div style={{ fontFamily: 'sans-serif', padding: 24 }}>
      <Greeting name="World" />
      <p style={{ color: '#6b7280', fontSize: 14, marginTop: 4 }}>
        Try changing the name prop above, or click the counter.
      </p>
      <Counter />
    </div>
  )
}`} />

## Summary

<Callout type="success" title="What we learned">
- React is a **declarative, component-based JavaScript library** for building UIs
- You describe **what** the UI should look like — React handles the DOM updates
- It was created by Facebook and is now the **most widely used UI library** in the world
- React is a **library** (not a framework) — it handles only the View layer
- It uses a **Virtual DOM** to efficiently update the real DOM
</Callout>
