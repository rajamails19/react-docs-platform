With stable keys, React can efficiently update only the changed items. Keys must be: unique among siblings, stable (don't use array index if the list is reordered or filtered — use an ID from the data), and strings or numbers. ## Summary

<Callout type="success" title="Key Takeaways for Junior Interviews">
- Know the React mental model: UI = f(state). Components render based on props and state.
- Understand the difference: props (read-only, from parent), state (mutable, internal)
- Know the three useEffect dependency patterns and when to use each
- Be able to explain why keys are needed in lists (not just "React needs it")
- Know the Rules of Hooks and why they exist (consistent hook call order)
</Callout>
