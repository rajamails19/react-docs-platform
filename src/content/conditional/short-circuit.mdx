# Short-Circuit Evaluation

Short-circuit evaluation lets you conditionally render UI with minimal code. Understanding how JavaScript's logical operators behave is essential for reading and writing React code.

## Logical AND (`&&`)

```
A && B → if A is truthy, returns B; otherwise returns A (short-circuits)
```

```tsx
// Show badge only when there are notifications
<div>
  <Bell />
  {notifications.length > 0 && (
    <span className="badge">{notifications.length}</span>
  )}
</div>

// Show admin panel only for admins
{user.isAdmin && <AdminPanel />}

// Loading spinner
{isLoading && <Spinner />}
```

## Logical OR (`||`) — Default Values

```
A || B → if A is truthy, returns A; otherwise returns B
```

```tsx
// Show user's name or a fallback
<h1>{user.displayName || 'Anonymous'}</h1>

// Show custom message or default
<p>{errorMessage || 'Something went wrong'}</p>
```

## Nullish Coalescing (`??`) — Null/Undefined Only

`??` is stricter than `||` — it only falls back when the left side is `null` or `undefined` (not `0`, `''`, or `false`):

```tsx
// ❌ || treats 0 as falsy — wrong for counts
<span>{itemCount || 'No items'}</span>   // shows "No items" when count is 0

// ✅ ?? only falls back on null/undefined
<span>{itemCount ?? 'No items'}</span>   // shows 0 correctly

// ✅ Useful for optional config values
const timeout = config.timeout ?? 5000
```

## Optional Chaining (`?.`) in JSX

Access deeply nested values safely:

```tsx
// Without optional chaining — crashes if user is null
<p>{user.profile.avatar.url}</p>

// With optional chaining — returns undefined safely
<p>{user?.profile?.avatar?.url ?? 'No avatar'}</p>

// Conditionally render nested data
{user?.subscription?.plan && (
  <Badge>{user.subscription.plan}</Badge>
)}
```

## Common Patterns Combined

```tsx
function UserHeader({ user }: { user: User | null }) {
  return (
    <header>
      {/* Render nothing if no user */}
      {user && (
        <div>
          {/* Avatar or fallback */}
          <img src={user.avatar ?? '/default-avatar.png'} alt={user.name} />

          {/* Name with fallback */}
          <h2>{user.displayName || user.email}</h2>

          {/* Only show badge if pro */}
          {user.plan === 'pro' && <ProBadge />}

          {/* Unread count — ?? protects against 0 bug */}
          {(user.unreadCount ?? 0) > 0 && (
            <span>{user.unreadCount} unread</span>
          )}
        </div>
      )}
    </header>
  )
}
```

## Summary

<Callout type="success" title="What we learned">
- `&&` renders the right side only when the left side is truthy — use `count > 0 &&` not `count &&` to avoid rendering `0`
- `||` returns the first truthy value — good for string/value fallbacks
- `??` only falls back on `null`/`undefined` — use for numeric values and optional config
- `?.` optional chaining safely accesses nested properties — pair with `??` for defaults
</Callout>
