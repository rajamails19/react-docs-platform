# The Children Prop

`children` is a special prop automatically passed to every component — it represents whatever is placed between the opening and closing tags of the component. It's the foundation of composition in React.

## Basic Usage

```tsx
interface CardProps {
  children: React.ReactNode   // anything React can render
}

function Card({ children }: CardProps) {
  return (
    <div className="card">
      {children}
    </div>
  )
}

// Usage — whatever you put between <Card> tags becomes children
<Card>
  <h2>Hello</h2>
  <p>This is inside the card.</p>
</Card>

// children can also be text
<Card>Just a string</Card>

// or nothing (null/undefined)
<Card />
```

## Why Children Matter

Without children, every variation of a component needs its own prop:

```tsx
// ❌ Rigid — you have to predefined every possible content
function Modal({ title, body, footer }: { title: string; body: string; footer: string }) {
  return (
    <div className="modal">
      <h2>{title}</h2>
      <p>{body}</p>
      <div>{footer}</div>
    </div>
  )
}

// ✅ Flexible — callers decide the structure
function Modal({ children }: { children: React.ReactNode }) {
  return <div className="modal">{children}</div>
}

// Now Modal works for any content:
<Modal>
  <ConfirmationForm onSubmit={handleSubmit} />
</Modal>
<Modal>
  <VideoPlayer src={url} />
</Modal>
```

## Multiple "Slots" Pattern

For structured layouts, use named props alongside children:

```tsx
interface PageLayoutProps {
  header:   React.ReactNode
  sidebar:  React.ReactNode
  children: React.ReactNode   // main content
  footer?:  React.ReactNode
}

function PageLayout({ header, sidebar, children, footer }: PageLayoutProps) {
  return (
    <div className="layout">
      <header>{header}</header>
      <div className="layout-body">
        <aside>{sidebar}</aside>
        <main>{children}</main>
      </div>
      {footer && <footer>{footer}</footer>}
    </div>
  )
}

// Usage
<PageLayout
  header={<TopNav />}
  sidebar={<SideNav links={links} />}
  footer={<Footer />}
>
  <ArticleContent />
</PageLayout>
```

## Inspecting Children: `React.Children`

```tsx
import React from 'react'

function Tabs({ children }: { children: React.ReactNode }) {
  const count = React.Children.count(children)

  return (
    <div>
      <p>{count} tabs</p>
      <div className="tabs">
        {React.Children.map(children, (child, index) => (
          <div key={index} className="tab">
            {child}
          </div>
        ))}
      </div>
    </div>
  )
}
```

## Render Props (Function as Children)

Children can also be a function — a pattern called "render props":

```tsx
interface MouseTrackerProps {
  children: (position: { x: number; y: number }) => React.ReactNode
}

function MouseTracker({ children }: MouseTrackerProps) {
  const [pos, setPos] = useState({ x: 0, y: 0 })

  return (
    <div onMouseMove={(e) => setPos({ x: e.clientX, y: e.clientY })}>
      {children(pos)}
    </div>
  )
}

// The caller controls what's rendered with the position
<MouseTracker>
  {({ x, y }) => <p>Mouse at ({x}, {y})</p>}
</MouseTracker>
```

<Challenge title="Build a multi-slot layout">
Add a `header`, `footer`, and `sidebar` prop to `Card` (alongside `children`):

```tsx
function Card({ header, children, footer }: {
  header: React.ReactNode
  children: React.ReactNode
  footer?: React.ReactNode
}) {
  return (
    <div>
      <div className="header">{header}</div>
      <div className="body">{children}</div>
      {footer && <div className="footer">{footer}</div>}
    </div>
  )
}
```

Usage: `<Card header={<h2>Title</h2>} footer={<button>Save</button>}>Content</Card>`

## Summary

<Callout type="success" title="What we learned">
- `children` is the content placed between a component's opening and closing tags
- Type it as `React.ReactNode` (accepts JSX, strings, numbers, arrays, null)
- Children enable composition — components become flexible containers
- Use named props alongside children for structured multi-slot layouts
- Children can be a function (render props pattern) for sharing stateful logic
</Callout>
