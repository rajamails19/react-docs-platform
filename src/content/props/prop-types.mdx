# Prop Types & TypeScript

Defining the shape of a component's props catches bugs at compile time, documents the API, and enables autocomplete. In modern React with TypeScript, you use `interface` or `type` instead of the legacy `PropTypes` library.

## The Problem Without Types

```tsx
// ❌ No type info — what does this component accept?
function UserCard({ name, age, isAdmin, onDelete }) {
  return <div>{name}</div>
}

// Caller has no idea what's valid:
<UserCard name={123} isadmin={true} onDlete={handleDelete} />
//        ^^^^^^^^^ wrong type  ^^^^^^^^^^^ typo   ^^^^^^^^^ typo — silent bug!
```

## TypeScript Interface (Recommended)

```tsx
// Define the shape with an interface
interface UserCardProps {
  name:      string
  age:       number
  isAdmin?:  boolean          // ? = optional, defaults to undefined
  onDelete:  (id: string) => void
  role:      'viewer' | 'editor' | 'admin'  // union = literal types
}

function UserCard({ name, age, isAdmin = false, onDelete, role }: UserCardProps) {
  return (
    <div>
      <h2>{name} ({role})</h2>
      <p>Age: {age}</p>
      {isAdmin && <span>Admin</span>}
      <button onClick={() => onDelete(name)}>Delete</button>
    </div>
  )
}

// ✅ TypeScript catches errors at compile time:
<UserCard
  name="Alice"
  age={30}
  role="editor"
  onDelete={(id) => console.log(id)}
/>
```

## Common Prop Type Patterns

```tsx
interface ExampleProps {
  // Primitives
  title:       string
  count:       number
  isVisible:   boolean

  // Optional with default
  variant?:    'primary' | 'secondary'   // optional union

  // Arrays
  items:       string[]
  users:       User[]

  // Objects
  config:      { timeout: number; retries: number }

  // Functions
  onClick:     () => void
  onChange:    (value: string) => void
  onSubmit:    (data: FormData) => Promise<void>

  // React-specific
  children:    React.ReactNode       // anything React can render
  icon:        React.ReactElement    // must be a JSX element
  style?:      React.CSSProperties   // inline style object
  className?:  string

  // Refs
  inputRef?:   React.RefObject<HTMLInputElement>
}
```

## `type` vs `interface`

```tsx
// interface — extendable, preferred for component props
interface ButtonProps {
  label: string
  disabled?: boolean
}

// Extend it for a specialized component:
interface IconButtonProps extends ButtonProps {
  icon: React.ReactNode
}

// type alias — preferred for unions, computed types
type ButtonVariant = 'primary' | 'secondary' | 'ghost' | 'danger'
type Size = 'sm' | 'md' | 'lg'

// Combine with intersection
type IconButtonProps = ButtonProps & { icon: React.ReactNode }
```

## Spreading Props (with Omit)

```tsx
// Accept all button HTML attributes + custom props
interface CustomButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant: 'primary' | 'secondary'
  isLoading?: boolean
}

function CustomButton({ variant, isLoading, children, ...rest }: CustomButtonProps) {
  return (
    <button
      {...rest}                  // spread all native button attrs (onClick, disabled, type, etc.)
      disabled={isLoading || rest.disabled}
      className={`btn btn-${variant}`}
    >
      {isLoading ? 'Loading...' : children}
    </button>
  )
}

// Usage — all native button attributes work automatically:
<CustomButton variant="primary" onClick={handleClick} type="submit" aria-label="Save">
  Save
</CustomButton>
```

## Legacy: PropTypes (Class Components / JS Projects)

```tsx
// Only needed for JavaScript projects (not TypeScript)
import PropTypes from 'prop-types'

function UserCard({ name, age }) {
  return <div>{name} ({age})</div>
}

UserCard.propTypes = {
  name: PropTypes.string.isRequired,
  age:  PropTypes.number,
}

UserCard.defaultProps = {
  age: 0,
}
```

<Callout type="warning" title="PropTypes are runtime-only">
PropTypes only validate at runtime in development — they don't catch errors at compile time. TypeScript catches errors before the code runs. In TypeScript projects, don't install `prop-types` — use interfaces.
</Callout>

## Summary

<Callout type="success" title="What we learned">
- Use `interface FooProps` or `type FooProps` to define prop shapes in TypeScript
- `?` marks optional props; provide defaults in the destructuring parameter
- `React.ReactNode` for children, `React.CSSProperties` for style, `React.ButtonHTMLAttributes` to extend native elements
- `interface` is extendable (good for props); `type` is flexible for unions/intersections
- Legacy `PropTypes` is runtime-only JavaScript — avoid in TypeScript projects
</Callout>
