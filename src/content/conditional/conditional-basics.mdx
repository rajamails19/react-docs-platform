# Conditional Rendering

React lets you render different UI based on conditions. Since JSX is just JavaScript, you use standard JS techniques — ternary operators, `&&`, and `if` statements outside JSX.

## Ternary Operator

The most common pattern for toggling between two elements:

```tsx
function Greeting({ isLoggedIn }: { isLoggedIn: boolean }) {
  return (
    <div>
      {isLoggedIn ? (
        <h1>Welcome back!</h1>
      ) : (
        <h1>Please sign in.</h1>
      )}
    </div>
  )
}
```

## Logical AND (`&&`)

When you want to render something OR nothing:

```tsx
function Notifications({ count }: { count: number }) {
  return (
    <div>
      <h1>Inbox</h1>
      {count > 0 && (
        <p>You have {count} new messages.</p>
      )}
    </div>
  )
}
```

<Callout type="warning" title="Avoid rendering 0">
`{count && <p>...</p>}` renders the number `0` when count is 0, because `0` is falsy but React renders it.

**Fix:** Use `{count > 0 && <p>...</p>}` or `{!!count && <p>...</p>}`.
</Callout>

## If Statement (Outside JSX)

For complex conditions, assign JSX to a variable before returning:

```tsx
function LoadingState({ status }: { status: 'loading' | 'error' | 'success' }) {
  if (status === 'loading') {
    return <Spinner />
  }

  if (status === 'error') {
    return <ErrorMessage />
  }

  return <Content />
}
```

## Switch Statement

Clean for multiple states:

```tsx
type Status = 'idle' | 'loading' | 'success' | 'error'

function StatusBadge({ status }: { status: Status }) {
  const badge = () => {
    switch (status) {
      case 'idle':    return <span className="badge-gray">Idle</span>
      case 'loading': return <span className="badge-blue">Loading…</span>
      case 'success': return <span className="badge-green">Done</span>
      case 'error':   return <span className="badge-red">Failed</span>
    }
  }

  return <div>{badge()}</div>
}
```

## Nullish Rendering

Return `null` to render nothing at all (no DOM output):

```tsx
function Alert({ message }: { message: string | null }) {
  if (!message) return null   // renders nothing

  return <div className="alert">{message}</div>
}
```

## Summary

<Callout type="success" title="What we learned">
- Ternary `condition ? A : B` — render one of two elements
- Logical AND `condition && <Element />` — render or nothing (watch out for `0`)
- `if` statements outside JSX — for complex multi-branch logic
- `switch` — clean multi-state rendering
- Return `null` to render nothing
</Callout>
